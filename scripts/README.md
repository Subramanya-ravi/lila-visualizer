# LILA BLACK – Player Journey Visualizer

A web-based tool for Level Designers to explore player behavior across LILA BLACK's maps — player paths, kill/death zones, loot hotspots, storm deaths, and heatmaps, with match playback.

**Live demo:** `<your-vercel-url>`

---

## Tech Stack

- **Frontend:** React 18 + Vite
- **Rendering:** Canvas 2D API
- **Data pipeline:** Python + PyArrow (parquet → JSON)
- **Hosting:** Vercel

---

## Setup

### 1. Prerequisites

- Node.js ≥ 18
- Python ≥ 3.9 with `pip`

### 2. Install Python dependencies

```bash
pip install pyarrow pandas
```

### 3. Run the preprocessor

Point it at the `player_data/` folder from the zip:

```bash
python scripts/preprocess.py --input ./player_data --output ./public/data
```

This generates:
- `public/data/index.json` — match list
- `public/data/matches/*.json` — per-match event files

### 4. Copy minimap images

Place the minimap images from the zip into `public/minimaps/`:

```
public/minimaps/
├── AmbroseValley_Minimap.png
├── GrandRift_Minimap.png
└── Lockdown_Minimap.jpg
```

### 5. Install JS dependencies & run

```bash
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173)

## Deployment (Vercel)

```bash
npm run build
npx vercel --prod
```

Or connect the repo to Vercel and it will auto-deploy on push.

**Important:** The `public/data/` folder (processed JSON) must be committed to the repo, or you must run the preprocessor as a build step. Recommended: commit the processed data.

If data files are large, add a Vercel build command:

```
# vercel.json
{
  "buildCommand": "pip install pyarrow pandas && python scripts/preprocess.py --input ./player_data --output ./public/data && npm run build"
}
```

## Features

| Feature | Description |
|---------|-------------|
| **Map selector** | Filter by AmbroseValley, GrandRift, or Lockdown |
| **Date + match filter** | Cascading dropdowns — pick date then match |
| **Player paths** | Per-player coloured trails on the minimap |
| **Bot paths** | Separate togglable layer for bot movement |
| **Event markers** | Kill (✦), Death (✕), Storm (◆), Loot (●) with distinct colours |
| **Human/bot distinction** | Humans = bright per-player colours; bots = grey |
| **Heatmap** | Density overlay of all player positions across the full match |
| **Timeline playback** | Scrub or play the match at 1×–100× speed |
| **Hover tooltip** | Mouse over any event marker to see event type + player ID |
| **Match stats** | Player count, bot count, duration, event breakdown in sidebar |
